# Step 4 — Add a Human

**Import a validation app and configure the human review task**

---

## Goal

Add a human validation step to the **Failed Match** path. When the agent flags a mismatch, a reviewer in **Action Center** sees the invoice and PO side by side and approves or rejects the match. The workflow resumes automatically based on the reviewer's decision.

## Humans in the Loop

Humans are most effective when they have all the information they need on one screen. In this step, you'll import a pre-built validation app that presents the agent's HTML comparison output to the reviewer — no switching between systems, no hunting for data.

## Steps

### Part 1: Import the validation app

1. In your workspace, navigate to **Apps** (or **Action Center Apps**).

2. Import the **2WM Validation App Template** from the shared templates library.

    ![Validation app template selected in shared templates](images/human-01.png){ .screenshot }

3. Review the imported app. It displays:
    - The side-by-side HTML comparison generated by the agent
    - The suggested rejection email
    - Two action buttons: **Approve** and **Reject**

    ![2WM Validation App interface](images/human-02.png){ .screenshot }

### Part 2: Configure the human validation task

4. In your **2-Way Matching Process** in Maestro, select the human task node on the **Failed Match** path.

5. Set the action type to **Action App Task**.

    ![Action App Task selected in task properties](images/human-03.png){ .screenshot }

6. Select the **2WM Validation App** you imported.

7. Set a descriptive task title, for example: **Invoice Review Required**.

8. Assign the task to yourself (or a specific user) so it appears in your Action Center queue.

    ![Task title set and user assigned](images/human-04.png){ .screenshot }

9. Map the agent outputs to the app inputs:

    | Agent Output | App Input |
    |-------------|-----------|
    | `out_DocumentsHTML` | Comparison HTML display field |
    | `out_SuggestedResponse` | Suggested rejection email field |

    ![Agent outputs mapped to app inputs](images/human-05.png){ .screenshot }

10. Save the task configuration.

    ![Completed task configuration](images/human-06.png){ .screenshot }

### Part 3: Configure the gateway decision logic

11. Add an **Exclusive Gateway** after the human task. This routes the workflow based on what the reviewer chose.

    ![Exclusive Gateway added after human task](images/human-07.png){ .screenshot }

12. Configure two outgoing paths:
    - **Approve** → proceed to payment
    - **Reject** → send supplier email

13. Set the expression for the **Approve** path:

    ```text
    vars.actionOutput == "Approve"
    ```

    ![Gateway expression for Approve path](images/human-08.png){ .screenshot }

### Part 4: Test the human-in-the-loop flow

14. Run the process in debug mode. Trigger a **Failed Match** scenario so the human task activates.

    ![Process debug showing human task triggered](images/human-09.png){ .screenshot }

15. Open **Action Center** and find your task. Review the comparison and click **Approve** or **Reject**.

    ![Action Center task with invoice comparison and action buttons](images/human-10.png){ .screenshot }

16. Return to the process and confirm it resumed and routed correctly based on your decision.

    ![Process continued after human decision](images/human-11.png){ .screenshot }
