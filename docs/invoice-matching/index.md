# Invoice Matching Automation

**2-Way matching with standard orchestration**

---

## Overview

In this exercise you'll build a comprehensive multi-component automation using **Maestro** to orchestrate AI agents, robots, human validators, and external APIs into a single end-to-end invoice matching process.

## Core Concept

> Agentic orchestration, implemented in Maestro, empowers enterprise customers to coordinate their long-running enterprise processes between humans, robots, and AI agents, working seamlessly across different systems of records and engagement.

Maestro lets you design, implement, monitor, operate, and optimize long-running processes at scale by combining three participant types:

| Participant | Role |
|-------------|------|
| **Robot** | Structured, rule-based task execution |
| **AI Agent** | Intelligent decision-making |
| **Human** | Judgment, validation, exceptions |

## Five-Step Implementation

Follow these steps in order to build the complete process:

1. [**Create BPMN Diagram**](create-bpmn.md) — Design the end-to-end process flow
2. [**Add a Robot**](add-robot.md) — Automate structured task execution
3. [**Add an Agent**](add-agent.md) — Add AI-driven decision-making
4. [**Add a Human**](add-human.md) — Configure validation and escalation checkpoints
5. [**Add API Calls**](add-api.md) — Connect to external systems to complete the workflow
