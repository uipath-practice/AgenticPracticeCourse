# 大功告成！

!!! tip "恭喜！"

    你已经从零构建了一个完整的智能体发票匹配流程。

---

## 你构建了什么

一切都按计划运转！流程现在完整了——它获取发票 PDF，用 IXP 提取并验证数据，把异常转给人工审核，发送拒绝邮件，并存储批准的记录以供付款处理。这一切都由 Maestro 编排。

使用 UiPath 平台服务从未如此轻松！

小结：

| 组件 | 角色 |
|-----------|------|
| **机器人** | 从 Storage Bucket 中获取发票 PDF |
| **AI 智能体** | 用 IXP 提取数据，查询采购订单，并执行双向匹配 |
| **人工任务** | 在 Action Center 中呈现不匹配摘要，供人审核 |
| **API 连接器** | 发送拒绝邮件，并把批准的记录存入 Data Fabric |

这是一个生产级的流程模式。同样的设计——机器人 → 智能体 → 人 → API——适用于现实世界里各种各样的文档处理场景。

## 下一步

### 1. 发布流程，并从 Maestro 仪表盘运行

你的流程现在已经完成。把它发布到你的“**Personal Workspace**”（个人工作区），然后像运行其他 UiPath 流程一样运行它——它会按正确的顺序自动触发机器人和智能体作业，只在必要时才让人参与。

- 在 **Studio Web** 中打开你的解决方案
- 发布到你的 **Personal Workspace**
- 从 **Orchestrator** 或 **Maestro** 触发一次运行
- 在 **Maestro** 仪表盘中端到端地查看执行轨迹：

## 持续迭代

**改进智能体提示词**

- 当前的提示词会漏掉一些边缘情况——例如相似但不完全相同的行项目描述。
- 修改系统提示词，用同样的发票重新测试。
- 观察匹配决策发生了怎样的变化。

**测试边界**

- 把 `in_FailureProbability` 设成不同的值，多运行几次。
- 故意提交一张缺少 PO ID 的发票，观察升级路径。

**扩展流程**

- 添加第二种文档类型（例如贷项通知单）。
- 为高金额的不匹配引入一条新的升级路径。
- 把批准路径连接到另一个下游系统。

你在这里练习的技能——提示词工程、智能体编排、人机协同设计、API 集成——是现代企业自动化的基石。继续实验。改改提示词。把东西弄坏，再修好。这样学到的才记得牢。
