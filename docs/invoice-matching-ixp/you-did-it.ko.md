# 해냈습니다!

!!! tip "축하합니다!"

    완전한 에이전틱 인보이스 매칭 프로세스를 처음부터 만들어 냈습니다.

---

## 무엇을 만들었나요

모든 것이 계획대로 동작합니다! 이제 프로세스가 완성되었습니다. 인보이스 PDF를 가져오고, IXP로 데이터를 추출·검증하고, 예외를 휴먼 검토로 라우팅하고, 반려 이메일을 보내고, 승인된 레코드를 지불 처리를 위해 저장합니다. 이 모든 것을 Maestro가 오케스트레이션합니다.

UiPath Platform 서비스를 활용하는 일이 이렇게 쉬웠던 적은 없습니다!

요약:

| 구성 요소 | 역할 |
|-----------|------|
| **로봇** | Storage Bucket에서 인보이스 PDF 가져오기 |
| **AI 에이전트** | IXP로 데이터를 추출하고, 구매 주문(PO)을 조회하고, 2-way 매칭 실행 |
| **휴먼 작업** | 검토를 위해 Action Center에 불일치 요약 표시 |
| **API 커넥터** | 반려 이메일을 보내고 승인된 레코드를 Data Fabric에 저장 |

이것은 프로덕션 패턴의 프로세스입니다. 로봇 → 에이전트 → 사람 → API로 이어지는 이 설계는 다양한 실제 문서 처리 시나리오에 그대로 적용됩니다.

## 계속 다듬어 보세요

**에이전트 프롬프트 개선하기**

- 현재 프롬프트는 몇 가지 엣지 케이스를 놓칩니다. 예를 들어 비슷하지만 완전히 같지는 않은 라인 아이템 설명 같은 경우입니다.
- 시스템 프롬프트를 수정하고 같은 인보이스로 다시 테스트해 보세요.
- 매칭 결정이 어떻게 달라지는지 측정해 보세요.

**한계 시험해 보기**

- `in_FailureProbability`를 여러 값으로 바꿔 가며 여러 번 실행해 보세요.
- PO ID가 빠진 인보이스를 일부러 제출하고 에스컬레이션 경로를 관찰해 보세요.

**프로세스 확장하기**

- 두 번째 문서 유형(예: 대변전표) 추가하기.
- 고액 불일치를 위한 새로운 에스컬레이션 경로 도입하기.
- 승인 경로를 또 다른 다운스트림 시스템에 연결하기.

여기서 연습한 기술 — 프롬프트 엔지니어링, 에이전틱 오케스트레이션, 휴먼 인 더 루프 설계, API 통합 — 은 현대 엔터프라이즈 자동화의 기본 재료입니다. 계속 실험하세요. 프롬프트를 바꿔 보세요. 망가뜨려 보세요. 고쳐 보세요. 그래야 진짜 자기 것이 됩니다.
