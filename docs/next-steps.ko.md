# 에이전틱 자동화 워크숍에 참여해 주셔서 감사합니다!

**UiPath Platform 서비스를 활용하는 일이 이렇게 쉬웠던 적은 없습니다!**

---

## 해냈습니다!

첫 번째 에이전틱 오케스트레이션 프로세스가 완성되었습니다! 이제 개인 워크스페이스에 게시하고, UiPath Platform의 다른 프로세스와 똑같이 실행할 수 있습니다. 올바른 순서로 로봇 작업과 에이전트 작업이 자동으로 트리거되고, 꼭 필요할 때만 사람이 개입합니다.

게시한 프로세스를 몇 번 실행한 뒤 Maestro에서 각 트랜잭션의 실행 상세 정보를 살펴보세요. 분석 화면을 통해 프로세스가 처음부터 끝까지 어떻게 흘러가는지 한눈에 파악할 수 있습니다.

결과는 이런 모습이어야 합니다.

이 실습들을 완료하면서 다음 요소를 결합한 프로덕션 패턴의 자동화를 만들었습니다.

| 구성 요소 | 역할 |
|-----------|------|
| **로봇** | 구조화된 결정적 작업 실행 |
| **AI 에이전트** | 지능적인 의사 결정과 데이터 처리 |
| **사람** | 판단, 검증, 예외 처리 |
| **API** | 시스템 통합과 데이터 교환 |

---

## 바로 이어서 할 일

### 1. 프로세스 게시하고 실행하기

완성한 프로세스를 개인 워크스페이스에 게시한 다음, 다른 UiPath 프로세스와 마찬가지로 실행해 보세요.

- 워크스페이스에서 Maestro 열기
- 프로세스 게시하기
- 샘플 데이터로 테스트 실행 트리거하기
- 실행 트레이스 확인하기

### 2. Maestro 분석 살펴보기

몇 번 실행하고 나면 Maestro 분석 기능을 사용할 수 있습니다.

- 트랜잭션 단위 실행 상세 정보
- 엔드투엔드 프로세스 흐름 시각화
- 병목 지점 식별
- 예외 및 에스컬레이션 비율

**Maestro → Analytics**를 열어 프로세스 성능을 살펴보세요.

## 계속 다듬어 보세요

진짜 배움은 실험에서 나옵니다.

**프롬프트 엔지니어링**

- 에이전트의 시스템 프롬프트 수정해 보기
- 변경이 분류나 매칭 결정에 어떤 영향을 주는지 테스트하기
- 엣지 케이스를 처리하는 다양한 접근 시도해 보기

**한계 시험해 보기**

- 에이전트에게 특이하거나 모호한 입력 넣어 보기
- 어디서 실패하거나 예상 밖의 결과가 나오는지 확인하기
- 그런 실패 모드까지 감안해 설계하기

**프로세스 확장하기**

- 매칭 프로세스에 새 문서 유형 추가하기
- 새로운 에스컬레이션 경로 도입하기
- 시스템 하나 더 연결하기
