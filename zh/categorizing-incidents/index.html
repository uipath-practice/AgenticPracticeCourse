    技术支持事件分类¶

    构建一个智能的 ServiceNow 事件分诊智能体，实现工单自动路由

    概览¶

    UiPath Agent Builder 是一款重要工具：即使几乎没有编码经验，你也能用它开发智能自动化解决方案。它让创建、设计和部署智能体变得简单——这些智能体可以调用自动化、使用 API，并与其他系统无缝协作。

    通过提供低代码环境，Agent Builder 让自动化能力人人可及：无论是技术用户还是非技术用户，都能打造强大的、由上下文驱动、贴合自身业务需求的智能体。

    在本练习中，你将使用 UiPath Agent Builder 创建一个自动分类并路由 ServiceNow 事件的智能体。你会学到如何让决策扎根于真实数据、如何用自动化测试验证表现，以及如何把智能体连接到线上系统。

    整个练习分几步展开：

    • 首先，你会教智能体根据静态描述对事件进行分类，并用示例输入测试它，观察 LLM 的行为。

    • 接着，你会使用 Context Grounding（上下文关联）优化结果，并通过 Evaluations（评估）验证表现。

    • 然后，你会“让智能体长出双手”：通过 Integration Service 把它连接到 ServiceNow，让它能获取事件详情，并根据分析结果更新工单。

    • 最后，你会配置 Escalations（升级），让智能体把拿不准的案例通过 Action Center 转交给人来处理。

    练习结束时，你将拥有一个功能完整、可用于生产的分诊智能体。开始吧！

    步骤¶

    步骤 重点
    使用上下文的 LLM 让智能体扎根于真实事件数据，消除幻觉。创建一个分析事件描述的智能体，把决策锚定在结构化上下文上，并用评估集验证表现。
    工具与升级 把智能体连接到 ServiceNow 的实时 API，并实现升级工作流。从真实系统获取事件详情，用分类决策更新工单，并通过 Action Center 把含糊不清的案例转交给人工审核者。

    训练环境

    登录 cloud.uipath.com/tpenlabs，并记得在本练习中使用租户 AgenticPractice。
